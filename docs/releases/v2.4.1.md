# v2.4.1

**Released:** 2026-04-24  
**Type:** Patch — documentation and export pipeline hardening

## What's new

### Datadog export pipeline

- **Full `DatadogExporter` implementation** — metric series submission, dashboard export, monitor export (single and bulk), with retry-aware HTTP client and exponential backoff ([PR #14](https://github.com/org-beaconstone/titan/pull/14))
- **Datadog authentication module** — `DatadogAuthConfig`, `build_auth_headers()`, startup validation via `validate_auth_config()` ([PR #14](https://github.com/org-beaconstone/titan/pull/14))
- **TTL in-memory cache** — `DatadogExporter` now caches monitor and dashboard results for 300 s by default, reducing Datadog API call volume under high job load; stale cache returned gracefully on 429 ([PR #21](https://github.com/org-beaconstone/titan/pull/21))
- **GA readiness guards** — `_DW_DEPENDENCY` flag and `GABlockerError` in the export pipeline; `dw_dependency` field and `check_ga_readiness()` in the job queue; `ErrorClass.GA_BLOCKER` in error classification ([PR #22](https://github.com/org-beaconstone/titan/pull/22))

### Documentation

- `docs/datadog-export.md` — full reference doc: quick start, metric types, dashboard/monitor export, error handling, retry behaviour, caching ([PR #15](https://github.com/org-beaconstone/titan/pull/15))
- `docs/datadog-auth-examples.md` — setup guide for local dev, CI (GitHub Actions), and production (Docker/Kubernetes), with usage examples and troubleshooting ([PR #11](https://github.com/org-beaconstone/titan/pull/11))
- `docs/adr/0002-datadog-export-auth.md` — ADR documenting the API key + App key auth decision ([PR #15](https://github.com/org-beaconstone/titan/pull/15))
- `docs/adr/0003-datadog-export-strategy.md` — ADR comparing pull-on-demand, bulk scheduled sync, and webhook approaches; formalises TTL caching as the accepted next step ([PR #20](https://github.com/org-beaconstone/titan/pull/20))
- `docs/runbooks/titan-launch-readiness.md` — GA go/no-go criteria, DW-881 dependency checklist, QA gate thresholds, and escalation path ([PR #22](https://github.com/org-beaconstone/titan/pull/22))

## Bug fixes

None.

## Breaking changes

None. `DatadogExporter.__init__` gains an optional `cache_ttl` parameter (default 300 s); existing call sites are unaffected.

## Upgrade notes

Set the following environment variables before starting the exporter:

```bash
export DATADOG_API_KEY=your-api-key
export DATADOG_APP_KEY=your-app-key
# Optional — defaults to datadoghq.com
export DATADOG_SITE=datadoghq.com
```

See `docs/datadog-auth-examples.md` for full setup instructions.
